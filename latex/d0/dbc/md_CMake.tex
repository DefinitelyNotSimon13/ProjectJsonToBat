Grob gesagt ist CMake das System, das unser Projekt konfiguriert und "{}baut"{}. "{}\+Baut"{} ist hierbei in Anführungszeichen, da es nicht den klassichen Compiler wie bspw. gcc/g++ ersetzt. CMake stellt diesem in gewissermaßen die benötigten Anweisungen bereit, damit "{}komplexere"{} Projekte mit mehreren zu kompilierenden Dateien problemlos und konsistent gebaut werden können.\hypertarget{md_CMake_autotoc_md3}{}\doxysection{\texorpdfstring{CMake Grundlagen\+:}{CMake Grundlagen:}}\label{md_CMake_autotoc_md3}

\begin{DoxyCode}{0}
\DoxyCodeLine{\#\ CMakeLists.txt}
\DoxyCodeLine{}
\DoxyCodeLine{cmake\_minimum\_required(VERSION\ 3.22)}
\DoxyCodeLine{}
\DoxyCodeLine{project(ProjectName)}
\DoxyCodeLine{}
\DoxyCodeLine{add\_executable(ProjectName\ main.cpp)}

\end{DoxyCode}
 Das ist eine einfach CMake\+Lists.\+txt Datei. Diese stellt im Prinzip die Anweisung wie das Projekt aufgebaut ist bereit.
\begin{DoxyEnumerate}
\item Zuerst wird die minimal benötigte CMake Version gesetzt.
\item Dann wird das Projekt "{}initiert"{}. Hier können auch andere Parameter wie bspw. die Version mitgegeben werden.
\item Zuletzt wird die ausführbare Datei hinzugefügt.
\end{DoxyEnumerate}\hypertarget{md_CMake_autotoc_md4}{}\doxysection{\texorpdfstring{Weitere Anweisung}{Weitere Anweisung}}\label{md_CMake_autotoc_md4}

 Mit \textquotesingle{}set\textquotesingle{} kann man in CMake Variablen setzen. Bei uns ist das insbesodere relevant, da so alle .cpp Dateien gesammelt der Executable "{}gegeben"{} werden können. \begin{quote}
Wenn ihr also eine neue Klasse erstellt, muss die .cpp Datei dieser Klasse zu den SOURCES in CMake\+Lists.\+txt hinzugefügt werden. \end{quote}
Header die unter src/include liegen, werden automatisch erkannt.
